# Excel Export Uses Pluggable Table Actions With a Client-Side Handler

Excel export is a Table Action in `ng-advanced-table-ui`, not a core table feature, because it composes with toolbar controls but is still optional companion behavior. The `natTableExportExcel` directive resolves its table controller through the shared UI control pattern, exports all client-held rows and visible exportable leaf columns by default, lets applications replace the operation with an `exportHandler`, and exposes `export()` so custom handlers can still delegate to the client-side export. The directive listens for DOM `click` and keyboard activation by default, and exposes `#excelExport="natTableExportExcel"` plus `trigger(...)` so custom elements can bridge custom activation events explicitly. The client-side handler uses a direct `xlsx` dependency so the directive works automatically, while API-backed or custom workbook generation remains available through the handler hook.

The directive is intentionally not backed by an Angular `resource`. Excel export is a user-triggered side effect, not reactive async data fetching, and the directive should not own retained value/error/status state for consumer-provided handlers. It only guards duplicate activation while the handler promise is pending.
